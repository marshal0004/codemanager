# Rent A Helper - Complete Commercial Rental Service Documentation
*Student-Focused Time-Based Rental Marketplace*

---

## 📋 Project Overview

### App Concept: "Rent A Helper"
A commercial rental service platform where users can **rent skilled people for specific time periods**. Primary focus on students who can earn side income by renting out their time/skills and students who need temporary help.

### Core Value Proposition
**"Rent Skills by the Hour"** - Users can rent:
- **Note Makers**: Rent someone to take notes, create study guides (by hour/day/week)
- **Developers**: Rent backend/frontend developers for projects (by hour/project)
- **Researchers**: Rent research assistants for academic work
- **Friends/Companions**: Rent a friend for events, study sessions, social activities
- **Boyfriends/Girlfriends**: Rent companions for dates, events, social presence

## 🎯 Product Requirements Document (PRD)

### Project Vision
Create the **Airbnb of Skills** - A commercial platform where people rent out their time, skills, and companionship to others, with students as the primary target market for both earning and spending.

### Success Metrics
- **User Base**: 50,000+ registered users within 12 months
- **Rental Completion**: 90% successful rental completion rate
- **Revenue**: $100,000+ monthly GMV (Gross Merchandise Value)
- **Student Focus**: 70% of users are students
- **Repeat Usage**: 40% of users make multiple rentals

### Target Users

#### 1. Renters (People who need help/companionship)
- **Primary**: Students (18-28 years) needing academic help, social companionship
- **Secondary**: Young professionals, anyone needing temporary skilled help
- **Pain Points**: Lack of skills, social isolation, time constraints, project deadlines
- **Goals**: Get quality help, make social connections, complete projects

#### 2. Service Providers (People renting out their time/skills)
- **Primary**: Students looking for side income (18-28 years)
- **Secondary**: Young professionals, freelancers, skilled individuals
- **Pain Points**: Need flexible income, want to monetize free time
- **Goals**: Earn money, flexible work, meet new people, build experience

```sql
Provider_Profiles

├── bio (text)
├── skills (JSON array)
├── languages (JSON array)
├── hourly_rate (decimal)
```

#### Rental Bookings Table
```sql

├── booking_id (UUID, Primary Key)

├── booking_type (hourly/daily/weekly/monthly)

├── end_datetime
├── duration_hours

├── hourly_rate

├── payment_processing_fee (3%)
├── total_amount_paid
├── provider_earnings

├── cancellation_reason
├── special_requirements (JSON)
├── created_at
├── confirmed_at
├── completed_at
└── cancelled_at

Status Flow:
requested → confirmed → in_progress → completed
```

#### Reviews & Ratings Table
```sql
Reviews
├── review_id (UUID, Primary Key)
├── booking_id (Foreign Key)
├── reviewer_id (Foreign Key → Users)

Two-way reviews: Both renter and provider review each other
```

Step 2: Registration Process
Sarah's Actions:

Fills out information:
- Email: sarah.johnson@email.com
- Phone: (555) 123-4567
- Name: Sarah Johnson

Frontend Flow:
Registration Form Submission
├── Client-side validation
│   ├── Email format check
│   ├── Phone number format
│   ├── Required fields check
│   └── Age verification (18+)
├── Loading spinner activation
├── API call to backend
└── Handle response (success/error)
Backend Authentication Flow:
POST /api/auth/register
├── Input validation & sanitization
├── Check if email/phone already exists
├── Hash password using bcrypt
├── Generate unique user_id (UUID)
├── Save to Users table in PostgreSQL
├── Send SMS OTP via Twilio
├── Send welcome email via SendGrid
├── Generate temporary JWT token
└── Return response with token
Database Operations:
PostgreSQL - Users Table Insert:
{
  user_id: "uuid-12345",
  email: "sarah.johnson@email.com",
  phone: "+15551234567",
  first_name: "Sarah",
  last_name: "Johnson",
  age: 20,
  location: "Austin, TX",
  user_type: "renter",
  verification_status: "pending",
  created_at: "2024-01-15T10:30:00Z"
}

Redis Cache:
