🎯 Authentication Process Step-by-Step
Step 1: Popup Opens

popup.html loads
popup.js initializes
Checks chrome.storage.local for existing token
Shows login form OR authenticated view

Step 2: User Login

User enters credentials in popup.html
popup.js handles form submission
Makes API call to Flask server /auth/login
On success: stores token in chrome.storage.local

Step 3: Background Processing

popup.js sends AUTH_SUCCESS message to background.js
background.js updates global auth state
Initializes WebSocket connection with token
Broadcasts auth state change to all tabs

Step 4: Cross-Tab Sync

auth-content-script.js receives auth state update
Updates any auth-dependent UI on web pages
All tabs now reflect authenticated state

Step 5: Persistent Session

Token stored in chrome.storage.local persists across browser sessions
On extension startup, background.js validates stored token
Automatic re-authentication if token is valid


🔧 Flask Server Authentication Endpoints
Corresponding Server Files:

flask-server/app/routes/auth.py - Authentication API endpoints
flask-server/app/models/user.py - User model and authentication methods
flask-server/app/utils/validators.py - Input validation for auth



🔄 How It Works:
Login Flow:
User Input → popup.js → Flask API → Store Token → background.js → WebSocket → Sync All Tabs
Persistent Session:
Extension Startup → Check Storage → Validate Token → Auto-Login → Maintain State

Authentication Process:

Extension Popup: User clicks extension icon, popup.html loads

Login Interface: popup.js renders login form within extension popup

API Communication: Extension sends credentials to Flask server via HTTP/WebSocket

Token Response: Server returns JWT token instead of setting cookies

Background Persistence: background.js stores JWT token in extension storage

Sync State: Extension maintains authentication state across browser sessions

🔄 AUTHENTICATION CONNECTION FLOW

Flask Server ↔ Background.js ↔ Popup.js Triangle:

1. Initial Authentication (Extension → Server):

popup.js captures user credentials

Sends POST request to Flask /auth/login endpoint

Server validates and returns JWT token

background.js receives and stores token persistently

2. Background.js as Authentication Broker:

Acts as intermediary between popup and server

Manages token storage and refresh cycles

Handles authentication state for all extension components

Maintains WebSocket connections with authenticated sessions

3. Popup.js Authentication State:

Queries background.js for current auth status

Updates UI based on authentication state

Handles logout by messaging background script

Displays user info and authentication controls
